TRNSYS 16 Release notes
Version 16.00.0034
Please note that printed documentation is not delivered with this release. A provisional documentation set is available in electronic form (use the "TRNSYS 16 manual" shortcut created during installation or browse to %TRNSYS16%\Documentation\Trnsys16-MainMenu.pdf). The TRNSYS developers are working on some remaining program and documentation issues, and the documentation will be updated with first user feedback. Therefore, the printed version of the manual will be shipped with the first update of TRNSYS.

Please also note that distributors have different policies regarding printed manuals. Do not hesitate to contact your distributor directly if you have any additional question.

1. Known issues with the Source Code
Issue Nb Description
1002 Type 15 has been removed. That Type was not available through IISiBat  in TRNSYS 15 
1003 Type 26 has been removed. That Type is obsolete. An empty routine has been added so projects can be run but no output is produced
1006 Type 61 (call to ExtDll) conflicts with the new DLL-based architecture and has been discontinued. If you were using Type 61, please upgrade your component to TRNSYS 16 and compile it into an external DLL as any other Type
1008 Type 109 has not been fully updated to the TRNSYS 16 standard. One side effect is that outputs are always 0 at the initial time step
1010 Type 28: Printing of 12 summaries at a time (mode 2) to the listing file might beinterrupted if other components print to the lst file. Using Type 28 to output results to the lst file is not recommended, you should use n external file
1046 It is currently not possible to control Type 25, 28 and 65 printed output format. The number of printed digits has been increased since TRNSYS 15 and the new format is not always easy to read. It will import correctly in most spreadsheet programs such as MS Excel
1049 Type 60 has been enhanced with regard to heat transfert coefficients. However, there mey still be some issues with incorrect energy balances with long time steps
1056 Check statements have been made more flexible and are now handled in the kernel for any Type. However, no results are currently printed when the balance is closed. An warning is generated if the balance does not close
1060 Type 66 (calling EES) does not close EES at the end of the simulation, even if no error occurred
1061 The new equation processor does not allow variable names starting with 'EQU' , 'CON', 'INC' or 'END'. If such a variable exists, it will cause errors in processing the deck but the error message does not indicate the real cause of the problem
1063 Type 90 (wind turbine) has been merged with Type 190. However, both Types are not using the same data file format. Type 190 format has been kept. See the Wind Diesel example for an example of data file
1065 In case of errors in the simulation, TRNSYS does not always close external programs (e.g. EES, Matlab, Excel) properly. You may have to close the programs or terminate processes manually

Issue Nb Description
2037 While every effort has been made to ensure backwards compatibility, problems may occur when updating existing IISiBat projects or importing TRNSYS 15 deck files. Please contact your distributors in such cases
2044 Added layers do not immediately show up in a project. You have to close and re-open the project
2045 Dimensionless variables are handled as variables with "any" dimensions by the connection filter
2094 Not all supported modes of Type 89 are available as a proforma
2095 Not all supported modes of Type 33 are available as a proforma
2136 Type 50 and 94 proformas have detailed names for variables but no information when you click on "More"
2142 The TPF file format has been changed in this release, which means that you cannot open projects created with 16.00.0034 in previous versions of the Studio. This applies to all beta releases of TRNSYS 16. The Studio projects cannot be opened in IISiBat either. The new file format is backwards compatible, i.e. you can open files created with all previous versions of the Studio and IISiBat.
